XAJAX-Framework

Beim Surfen habe ich grade XAJAX entdeckt und es mir gleich mal genauer angeguckt: Das schöne daran ist, dass es gar nicht viel zum angucken gibt, aber dazu später mehr ;)

Bei XAJAX handelt es sich um eine PHP-Klassenbibliothek, die einem die meiste JavaScript-Arbeit bei der Entwicklung von AJAX-Anwendungen abnimmt: Einfach eine Datei ins PHP-Script einbinden, Objekt erzeugen und schon kann man seine PHP-Funktionen schreiben, die für die dynamischen Inhalte sorgen (hier zum Angucken ein Link ins XAJAX-Wiki: Learn XAJAX in 10 minutes).

Der Code kann somit in PHP geschrieben werden, um die Umsetzung in JavaScript kümmert sich dann XAJAX. Ich hab mir vor längerer Zeit mal andere PHP-Frameworks für AJAX angeguckt (SAJAX und noch irgendeins), fand diese aber nicht komfortabel - XAJAX hingegen scheint echt was Feines zu sein. Einziges Manko was mir bisher aufgefallen ist: Der erzeugte JavaScript-Code muss im Header der Seite eingebunden werden, hier wäre vielleicht eine Möglichkeit zum Auslagern in eine externe Datei besser.

Whatever, das kommt auf jeden Fall nächstes Mal zum Einsatz, bin gespannt :)
