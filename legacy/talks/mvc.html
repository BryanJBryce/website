Das MVC-Architekturmuster

Bremen Inside, 09.02.2007

Was ist MVC?

  • Entwurfsmuster für Software-Architektur
  • Jede interaktive Anwendung besteht aus Eingaben,
    einem Datenmodell und der Darstellung der Daten
  • Drei Komponenten:
    • Model: Anwendungslogik / Datenquelle (Zustand)
    • View: Datenausgabe für den Benutzer
    • Controller: Funktionsschnittstelle / Anwendungssteuerung
  • Beispiel: Wahlergebnisse

Model - View - Controller

  • Model:
    • weiß nichts vom View und Controller
    • stellt Operationen bereit, die der Controller nutzen kann
    • enthält die Programmlogik
  • View:
    • Präsentation der Daten
    • ruft Informationen aus dem Model ab
    • welcher View angezeigt wird, entscheidet der Controller

Model - View - Controller

  • Controller:
    • definiert den Zugriff auf das Model
    • ordnet Benutzerinteraktion Operationen im Model zu
    • sollte keine Anwendungslogik beinhalten
  • Vorteil: Entkopplung

Vorteile durch Entkopplung

  • Alle Teile sind unabhängig und austauschbar
  • Erhöhte Übersichtlichkeit > Reduzierung der Komplexität
  • Wiederverwendbarkeit der Komponenten
  • Erleichtert uns die Aufgabenteilung im Team

Beispiel: Webanwendung
